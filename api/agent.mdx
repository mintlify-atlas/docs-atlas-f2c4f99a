`@expect/agent` is the internal package that connects `expect-cli` to Claude Code and Codex CLI via the [Agent Communication Protocol (ACP)](https://agentclientprotocol.org). It exposes an Effect-based `Agent` service that streams session updates from whichever agent backend is selected.

<Note>
  `@expect/agent` is an internal package (`"private": true`) used by `expect-cli`. It is not published to npm and is not intended for direct use outside this monorepo. To run browser tests with an AI agent, use the `expect-cli` command directly.
</Note>

## Exported API

```typescript
// Core service
export { Agent, type AgentBackend } from "./agent";
```

## `AgentBackend`

The union type for supported agent backends:

```typescript
type AgentBackend = "claude" | "codex";
```

Pass this to `--agent` on the CLI or to `Agent.layerFor` when constructing an `Agent` layer.

---

## `Agent` service

An Effect service that wraps Claude Code and Codex CLI via ACP. The service provides two operations:

```typescript
class Agent extends ServiceMap.Service<
  Agent,
  {
    readonly stream: (
      options: AgentStreamOptions,
    ) => Stream.Stream<AcpSessionUpdate, AcpStreamError | AcpSessionCreateError | ...>;

    readonly createSession: (
      cwd: string,
    ) => Effect.Effect<SessionId, AcpSessionCreateError | ...>;
  }
>()("@expect/Agent") {
  static layerClaude: Layer;   // Claude Code backend
  static layerCodex: Layer;    // Codex CLI backend
  static layerFor: (backend: AgentBackend) => Layer;  // select by name
}
```

### Layers

<ParamField path="Agent.layerClaude" type="Layer">
  Provides the `Agent` service backed by Claude Code. Requires `claude login` to be completed.
</ParamField>

<ParamField path="Agent.layerCodex" type="Layer">
  Provides the `Agent` service backed by Codex CLI. Requires `OPENAI_API_KEY` or `codex login`.
</ParamField>

<ParamField path="Agent.layerFor(backend)" type="Layer">
  Selects the correct layer based on an `AgentBackend` string: `"claude"` → `layerClaude`, `"codex"` → `layerCodex`.
</ParamField>

## `AgentStreamOptions`

Options for the `Agent.stream` call:

```typescript
class AgentStreamOptions extends Schema.Class<AgentStreamOptions>("AgentStreamOptions")({
  cwd: Schema.String,
  sessionId: Schema.Option(Schema.String),
  prompt: Schema.String,
  systemPrompt: Schema.Option(Schema.String),
  mcpEnv: Schema.optional(Schema.Array(McpEnvEntry)),
}) {}
```

<ParamField path="cwd" type="string" required>
  Working directory the agent operates in. Should be the root of the project under test.
</ParamField>

<ParamField path="sessionId" type="Option<string>">
  Resume a previous ACP session by ID. Pass `Option.none()` to start a fresh session.
</ParamField>

<ParamField path="prompt" type="string" required>
  The prompt to send to the agent — typically the test plan instructions generated by the supervisor.
</ParamField>

<ParamField path="systemPrompt" type="Option<string>">
  Optional system prompt override.
</ParamField>

<ParamField path="mcpEnv" type="McpEnvEntry[]">
  Environment variable entries to pass to the agent's MCP server environment.
</ParamField>

## `detectAvailableAgents()`

Detects which supported coding agents are installed on the current machine by running `which` (or `where` on Windows) for each agent binary.

```typescript
import { detectAvailableAgents } from "@expect/agent";

const agents = detectAvailableAgents();
// ["claude"] | ["codex"] | ["claude", "codex"] | []
```

Returns `SupportedAgent[]` — a subset of `["claude", "codex", "cursor"]`.

<Note>
  `detectAvailableAgents` is used by `expect-cli init` to validate that at least one supported agent is installed before proceeding with setup.
</Note>

## ACP session updates

The `Agent.stream` method emits `AcpSessionUpdate` events as the agent runs. These map to the domain events rendered by the CLI's TUI:

| Update type | Description |
| --- | --- |
| `agent_message_chunk` | Text output from the agent |
| `agent_thought_chunk` | Agent reasoning / thinking |
| `user_message_chunk` | User message echoed back |
| `tool_call` | Agent invoked a tool (Bash, Read, etc.) |
| `tool_call_update` | Progress or completion of a tool call |
| `plan` | Agent emitted a structured task plan |

## Error types

| Error class | When it is thrown |
| --- | --- |
| `AcpStreamError` | Unexpected error during the ACP stream |
| `AcpSessionCreateError` | Failed to create an ACP session |
| `AcpProviderNotInstalledError` | The selected agent CLI is not installed |
| `AcpProviderUnauthenticatedError` | The agent CLI is not authenticated (`claude login` / `codex login`) |
| `AcpProviderUsageLimitError` | The agent's API usage limit has been reached |
